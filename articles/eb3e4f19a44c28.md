# Abstract
- Android ndkのアプリの公開には16KBページサイズが必須。
- 16KBページサイズ対応方法
- 16KBページサイズ対応をチェックする方法

# 概要
https://developer.android.com/guide/practices/page-sizes?hl=ja&authuser=1#ndk-build
上記のAndroid公式ページに記載通りなんだけど、結局何をすればいいんだ？ってなる。

おそらく10年ぶりぐらいにアプリ公開しようとしたら、仕様というか公開手順やらルールがいろいろ変わっててかなりめんどいことになってた。プライバシーポリシーの必須化とかで、時間溶かすんだけどなかでも 16KBページサイズ対応 にハマってしまった。
なのでその話。

## android ndkアプリは16KBページサイズへの対応が必須
16KBページサイズへの対応をしなかった場合は、aab(apk)ファイルの登録に失敗します。
なのでやらないという選択肢がない。
失敗画面は以下の通り。(赤線のところ)
![](https://storage.googleapis.com/zenn-user-upload/01e88f8eb8bc-20251123.png)
※**4KB LOAD section alignment, but 16KB is required**が表示されている!!

## 16KBページサイズへの対応方法
ホントはndkバージョン28以上を使用すればいいだけらしいのだけど、とある事情からバージョン27を使う必要がありまして。
で、下記一行を追加すれば解決しました。

```diff CMakeLists.txt
cmake_minimum_required(VERSION 3.22.1)

project("videophotobook")

add_library(VUFORIA_LIBRARY SHARED IMPORTED)
set_target_properties(VUFORIA_LIBRARY PROPERTIES IMPORTED_LOCATION
        ${CMAKE_CURRENT_SOURCE_DIR}/../jniLibs/${ANDROID_ABI}/libVuforiaEngine.so)

add_library(${CMAKE_PROJECT_NAME} SHARED
        VuforiaController.cpp
        GLESRenderer.cpp
        GLESUtils.cpp
        Jni.cpp)

+ target_link_options(${CMAKE_PROJECT_NAME} PRIVATE "-Wl,-z,max-page-size=16384")
target_include_directories(${CMAKE_PROJECT_NAME} PUBLIC include)

target_link_libraries(${CMAKE_PROJECT_NAME}
        android
        log
        GLESv3
        VUFORIA_LIBRARY)
```

## 16KBページサイズ対応をチェックする方法
Android Studio起動 → ビルド → Anaylyze APKを実行する。
![](https://storage.googleapis.com/zenn-user-upload/78a48223b71e-20251123.png)
下記画面が表示される。
![](https://storage.googleapis.com/zenn-user-upload/20b25b0e2e96-20251123.png)
※**4KB LOAD section alignment, but 16KB is required**が表示されてないのでOK

# まとめ
ndkバージョン28以上でビルドしましょう。仕方ないときは、"target_link_options(${CMAKE_PROJECT_NAME} PRIVATE "-Wl,-z,max-page-size=16384")"を追記しましょう。
以上です。
